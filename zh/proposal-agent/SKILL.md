# SKILL: 论文复现方案规划

通过与用户交互，了解目标论文信息，拆解 R0 / R1 / R2 三级复现计划并输出为 Markdown 文件。

---

## 何时使用

当用户提到"复现方案"、"复现计划"、"论文复现"等需要规划复现步骤的场景。

## PDF 解析

如需解析 PDF，使用 `bohrium-pdf-parser`。解析前先检查工作空间中是否已有解析结果（`.md`、`.txt`、`.json`），避免重复解析。

---

## 会话流程

```
1. 询问用户要复现的论文（标题、链接或关键词）
2. 了解论文核心内容（方法、数据集、主要结果）
3. 拆解 R0 / R1 / R2 三级复现计划
4. 将计划保存为 Markdown 文件
5. 在交互界面向用户展示完整计划
```

---

## R0 / R1 / R2 三级定义

### R0 - 跑通代码

核心目标：跑通代码，不注重结果是否准确。

1. **能拿到数据** - 获取数据（可以是样例数据或子集），确认可加载、格式正确
2. **能构建可运行代码** - 环境搭建、依赖安装、代码能正常运行
3. **能输出正确格式** - 端到端跑通，输出格式符合预期（不要求数值正确）
4. **能计算评估指标** - 评估脚本能跑通，能算出数值（不要求达标）

验收标准：数据能拿到、代码能跑、输出格式正确、指标能算出。

### R1 - 复现论文结果

核心目标：复现论文中的实验结果（图表、数值、性能指标等）。

1. 准备完整数据集
2. 按论文设定配置超参数，完整训练或加载预训练权重
3. 逐项复现论文报告的实验结果
4. 对比论文数值，分析误差来源

验收标准：主要实验结果在合理误差范围内被复现。

### R2 - 深度复现

核心目标：尝试解决论文中仍存在的问题，实现未完成的改进。

1. 挖掘论文中的 Future Work
2. 分析现有方法局限，设计改进方案
3. 改进算法实现，尝试提升性能
4. 与 R1 基线对比，验证改进效果

验收标准：至少一个改进方向有实验结果，并与 R1 基线对比分析。

## 交互要求

| 信息 | 说明 | 必需 |
|------|------|------|
| 论文标题或链接 | 确定目标论文 | 是 |
| 论文核心方法 | 了解论文做了什么 | 是 |
| 代码仓库 | 是否有官方代码 | 推荐 |
| 数据集 | 使用了什么数据 | 推荐 |
| 用户目标深度 | 想做到 R0 / R1 / R2 哪级 | 可选 |

---

## 输出要求

### 保存路径

- 默认：`~/proposal-<paper-slug>.md`
- 如有论文专属目录：`~/paper-diagnosis-reports/<PaperName>/proposal.md`

### Markdown 模板

```markdown
# 论文复现计划

## 论文信息

- **标题**: <论文标题>
- **链接**: <论文链接>
- **核心方法**: <一句话概括>

---

## R0 - 跑通代码

**目标**: 跑通代码流程，不要求结果准确

| # | 任务 | 说明 |
|---|------|------|
| 1 | 获取数据 | <数据来源、获取方式、格式说明> |
| 2 | 构建可运行代码 | <环境搭建、依赖安装、框架版本> |
| 3 | 跑通流程并输出 | <入口脚本、运行命令、预期输出格式> |
| 4 | 计算评估指标 | <评估脚本、指标类型、确认能算出数值> |

**验收**: 数据能拿到、代码能跑、输出格式正确、指标能算出

---

## R1 - 复现论文结果

**目标**: 复现论文中的实验结果
**前置**: R0 通过

| # | 任务 | 说明 |
|---|------|------|
| 1 | 完整数据准备 | <完整数据集获取和预处理> |
| 2 | 模型训练/加载 | <训练配置、超参数、预训练权重> |
| 3 | 逐项复现结果 | <要复现的具体 Table/Figure/指标列表> |
| 4 | 误差分析 | <结果对比、误差来源分析> |

**验收**: 主要实验结果在合理误差内与论文一致

---

## R2 - 深度复现

**目标**: 解决论文中仍存在的问题，尝试改进
**前置**: R1 通过

| # | 任务 | 说明 |
|---|------|------|
| 1 | 挖掘改进方向 | <论文 Future Work / Limitation 中的具体方向> |
| 2 | 设计改进方案 | <改进思路和实现方案> |
| 3 | 实现与实验 | <代码修改、实验配置> |
| 4 | 对比分析 | <与 R1 基线对比，量化改进效果> |

**验收**: 至少一个改进方向有实验结果并与基线对比
```

### 展示

文件保存后，将完整 Markdown 内容输出到交互界面。

---

## 约束

- 只做规划，不执行实际复现操作
- 不外泄用户私有数据
